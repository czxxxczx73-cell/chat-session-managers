# Contributing

1. Keep each change focused.
2. Never include real conversation data, credentials, usernames, or personal paths.
3. Do not change the existing `index.html` card layouts unless a maintainer explicitly approves a UI release.
4. Run `make check test package` on macOS before opening a pull request.
5. Update English and Simplified Chinese documentation together.
6. Do not add analytics, external content, outbound networking, or automatic destructive synchronization.
